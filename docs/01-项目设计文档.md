# 历史看板项目设计文档（V1）

## 1. 项目背景

面向“快速历史查询 + 中外同期对照”场景，提供 H5 应用。第一阶段聚焦中国历史基础数据与可用检索体验。

## 2. 设计原则

1. **可查询**：支持广义关键词检索，尽量避免“查不到”。
2. **可理解**：搜索结果不仅返回事件，还给出背景与影响。
3. **可对照**：自动给出同期世界事件，帮助建立全局历史观。
4. **可进化**：数据源、检索策略、展示能力可独立升级。
5. **Java 优先**：服务端与数据处理优先 Java 实现。

## 3. 总体架构

```text
[H5 前端]
   |
   | HTTP/JSON
   v
[Java API 服务 (Spring Boot)]
   |- 搜索服务（相关性评分）
   |- 事件详情服务
   |- 同期世界对照服务
   |- 时间带与发现服务
   |
   v
[结构化数据层]
   |- china_history_events.json
   |- world_history_events.json

[数据构建层]
   |- Java Dataset Builder (优先)
   |- Python Builder/Collector (兜底)
```

## 4. 模块设计

- 前端模块
  - 搜索输入区
  - 结果列表区
  - 详情与同期对照区
  - 时间带区
- 服务端模块
  - `HistorySearchService`：分词/归一化/打分
  - `HistoryContextService`：同期世界事件计算
  - `HistoryController`：统一 API
- 数据模块
  - Raw Seed（人工维护、可审校）
  - Build Output（标准字段、直接查询）

## 5. 数据流

1. 维护原始种子数据（中国/世界）
2. 运行构建器生成标准数据（带关键词字段）
3. Java 服务启动加载标准数据并建立索引
4. 前端调用 API 完成检索与展示

## 6. 扩展点

- 检索层：拼音、同义词、分词器、向量检索
- 数据层：文献来源、可信度评级、关系边
- 展示层：双时间轴、关系图谱、专题模式
- 服务层：分页、多条件过滤、导出能力

## 7. 风险与应对

- 历史数据准确性：引入来源字段和审校流程
- 任意输入召回不足：多策略召回 + fallback 推荐
- 数据规模增长：引入全文索引（ES/Lucene）
- 跨语言维护成本：明确“Java 主线 + Python 兜底”边界
