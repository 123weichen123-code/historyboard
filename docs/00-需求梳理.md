# 历史看板应用需求梳理（V1）

## 1. 目标一句话

构建一个 H5 历史看板，用户输入任意关键词后可以快速定位到相关中国历史事件，并可对照同时期世界历史关键节点。

## 2. 核心业务目标

1. **快查**：任意输入尽量都能返回有价值结果（高召回 + 可解释）。
2. **对照**：中国事件与同期世界事件联动展示。
3. **可扩展**：后续可拓展到更多数据源、关系图谱、深度分析视图。
4. **工程可维护**：数据采集、清洗、服务查询与前端展示解耦。

## 3. 用户角色与场景

- 普通用户：快速了解某个人物/事件/朝代/年份。
- 学生用户：做时间线学习与中外对比。
- 内容创作者：快速获取历史选题线索。

## 4. 核心功能范围（V1）

- 关键词检索（人物、事件、朝代、年份、别名、标签）
- 搜索建议词（自动补全）
- 事件详情（摘要、影响、关键人物、标签）
- 同期世界关键节点对照
- 中国历史时间带浏览
- 发现页（默认推荐 + 快速探索）

## 5. 非功能要求

- 首屏响应快（< 2s，局域网/本地）
- 搜索可用性高（无命中时仍给推荐结果）
- 接口可扩展（支持分页/排序/多条件）
- 前端可维护（模块化 UI 区块，预留扩展位）

## 6. 技术策略（用户新增要求）

- **服务端功能优先使用 Java 实现**（Spring Boot）。
- **数据获取优先 Java 实现**（Java 数据构建器/抓取器）。
- 当 Java 抓取某数据源复杂度高或生态受限时，允许使用 Python 作补充抓取，并通过统一数据规范入库。
