# 前端设计文档（H5）

## 1. 技术栈

- 原生 HTML/CSS/JavaScript
- 面向移动优先，兼容桌面端
- 通过 REST API 与 Java 服务通信

## 2. 信息架构

1. 顶部 Hero 区
   - 产品标题
   - 一句话说明
   - 搜索框与操作按钮
   - 快速建议词
2. 主体 Dashboard
   - 左：搜索结果列表
   - 右：事件详情 + 同期世界事件
3. 下方 Time Strip
   - 中国历史时间带快速入口

## 3. 页面交互流程

1. 首次进入：调用 `/api/history/discover` + `/api/history/timeline`
2. 用户输入关键词：调用 `/api/history/search`
3. 选择某条结果：调用 `/api/history/events/{event_id}`
4. 点击时间带节点：复用搜索流程

## 4. 组件设计

- SearchBar
  - 输入、检索按钮、随机按钮
- SuggestionChips
  - 热门词与推荐词
- ResultList
  - 可点击事件卡片，支持选中高亮
- DetailCard
  - 事件核心信息
- WorldContextList
  - 同期世界事件卡片列表
- TimelineStrip
  - 时序入口按钮网格

## 5. 状态管理（轻量）

- `state.query`
- `state.results`
- `state.selectedId`
- `state.quickQueries`
- `state.timeline`

## 6. 可用性与视觉策略

- 强可读排版：标题与正文层级明显
- 搜索反馈即时：无结果状态、错误状态明确
- 时间与标签结构化展示，降低理解成本
- 保留扩展位：关系图、来源卡片、筛选器

## 7. 扩展规划

- 增加筛选面板（朝代、类别、年份范围）
- 增加“中外并行时间轴”可视化模式
- 增加事件关系图与来源引用区
