# 历史看板 PRD（V1）

## 1. 产品概述

- 产品名：史镜 HistoryBoard
- 形态：H5
- 目标：用户可通过任意关键词快速查询中国历史，并查看同期世界事件

## 2. 用户痛点

1. 信息分散：历史信息散落在百科、文章、视频中，检索成本高。
2. 时间感弱：很多用户只知事件，不清楚具体历史时序。
3. 对比难：缺少“同一时期世界发生了什么”的联动认知。

## 3. 产品目标与指标

- 目标 1：搜索可用性
  - 指标：常见关键词 Top100 命中率 > 95%
- 目标 2：检索速度
  - 指标：接口 P95 响应 < 500ms（本地/测试环境）
- 目标 3：对照能力
  - 指标：Top3 结果均可返回同期世界事件

## 4. 用户故事

- 作为学生，我输入“1911”后能看到辛亥革命及前后背景。
- 作为普通用户，我输入“秦始皇”可以直接看到统一六国等关键事件。
- 作为创作者，我希望快速获得同期世界大事用于内容对比。

## 5. 功能需求

### FR-01 搜索

- 输入：任意文本（事件、人物、朝代、年份、别名）
- 输出：按相关性排序的历史事件列表
- 规则：
  - 支持精确匹配 + 模糊匹配
  - 无命中时返回推荐结果（fallback）

### FR-02 建议词

- 输入：用户已输入前缀
- 输出：建议词列表

### FR-03 事件详情

- 输出：标题、朝代、时间、摘要、影响、人物、标签

### FR-04 同期世界事件对照

- 输入：中国事件起止年份
- 输出：时间窗口内世界事件列表

### FR-05 历史时间带

- 输出：可点击历史节点列表，支持二次检索

### FR-06 发现页

- 输出：默认推荐检索词 + 默认事件结果

## 6. 非功能需求

- NFR-01 可维护性：字段标准化，文档齐全
- NFR-02 可观测性：记录查询日志（后续）
- NFR-03 安全性：接口限流与输入校验（后续）
- NFR-04 国际化：前端文案与数据结构预留多语言能力（后续）

## 7. 版本范围

- V1 包含：静态数据、检索、对照、时间带、发现页
- V2 规划：数据来源引用、关系图谱、拼音/同义词、高级筛选
